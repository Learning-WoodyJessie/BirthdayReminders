# BirthdayReminders — AI Context

## What this project does
Automated birthday and anniversary reminder system. Runs daily via GitHub Actions.
Reads contacts from `data/people.yaml`, finds upcoming events, uses Claude to generate
personalised messages, and sends them via WhatsApp Cloud API.

## Key constraints
- **No database, no server** — all data lives in YAML files. Keep it that way.
- **No web app** — GitHub Actions is the scheduler/runner.
- **Secrets never in code** — all credentials are GitHub Secrets (see README).
- **OpenAI model**: always use `gpt-4o` for message generation.

## File map
| File | Purpose |
|---|---|
| `data/people.yaml` | Source of truth for all contacts and dates |
| `config.yaml` | Reminder lead times, owner WhatsApp number |
| `scripts/check_reminders.py` | Daily runner: finds events → generates messages → sends WhatsApp |
| `scripts/add_person.py` | Interactive CLI to add new contacts |
| `scripts/list_upcoming.py` | Preview upcoming events without sending anything |
| `.github/workflows/daily_reminder.yml` | GitHub Actions cron (8AM UTC daily) |
| `requirements.txt` | Python deps: anthropic, requests, PyYAML |

## Data schema (`people.yaml`)
```yaml
- name: string (required)
  relationship: string (required) — mother, best friend, colleague, etc.
  birthday: YYYY-MM-DD or --MM-DD (optional)
  anniversary: YYYY-MM-DD (optional)
  notes: string — hobbies, life events, shared memories (used by Claude)
  phone: E.164 string e.g. +14155550001 (optional — enables direct day-of message)
  groups: list of WhatsApp group IDs (optional — posts to groups on day-of)
```

## Message audiences
Three message types are generated by Claude, each with a different tone:
- `owner` — reminder framed for the owner ("don't forget, here's a message to send")
- `direct` — warm personal message sent directly to the person on day-of
- `group` — celebratory group-friendly message posted to WhatsApp groups on day-of

## GitHub Secrets required
| Secret | Description |
|---|---|
| `OPENAI_API_KEY` | OpenAI API key |
| `WHATSAPP_ACCESS_TOKEN` | Meta permanent access token |
| `WHATSAPP_PHONE_NUMBER_ID` | WhatsApp Business phone number ID |
| `MY_WHATSAPP` | Owner's WhatsApp number (E.164) |

## What NOT to change without careful thought
- The `generate_message` audience logic — the three-way split (owner/direct/group) is intentional
- YAML as data store — do not migrate to a database
- `days_until` logic handles year-boundary wrap correctly — don't simplify it
